# SSR-Verifier v3.0 — Technical Project Report
**Official Documentation for Jain (Deemed-to-be University)**

## 1. Project Overview
**SSR-Verifier** is a high-performance auditing platform designed to automate the Data Validation and Verification (DVV) process for NAAC accreditation. The system identifies detectable failures in Self Study Reports (SSR) that lead to score deductions, such as arithmetic inconsistencies, broken link protocols, and non-compliant domain hosting.

## 2. Core Features
### A. Intelligent Audit Engine (Powered by Groq Cloud AI)
The system has been upgraded to **v3.0**, integrating the **Groq Llama-3.3 Inference Engine**. 
- **High-Speed Analysis**: Sub-second semantic validation of complex accreditation documents.
- **Pattern Matching**: Detects "Metadata Drift" where document creation dates mismatch claimed historical events.
- **Compliance Mapping**: Automatically aligns institutional evidence against the latest 2025 NAAC Standard Operating Procedures (SOP).

### B. Secure Authentication & Identity
Built with a zero-trust approach to institutional data:
- **Google OAuth 2.0 Integration**: Seamless "Single Sign-On" for faculty and auditors.
- **Hardened Session Management**: Custom session layer using `connect-pg-simple` on **Neon Cloud DB**, configured with proxy-trust and cross-domain cookie security for stable production deployment.
- **Multi-Tenant Ready**: Support for local email/password login with bcrypt-secured hashing.

### C. Forensic Data Verifier
- **Arithmetic Forensics**: Full recalculation of extracted tables to detect manual "rounding fixes" or fabrications.
- **Domain Lockdown**: Automated enforcement of NAAC SOP requirements (denying `drive.google.com` or `dropbox.com` in favor of official `.ac.in` domain hosting).
- **OCR & Extraction**: Advanced PDF parsing that translates scanned certificates into structured, searchable data.

## 3. Technical Architecture (How We Built It)
We utilized a modern, scalable monolithic architecture to ensure low latency and high availability:

- **Frontend**: **React v18 + Vite** with **Tailwind CSS** for a premium, dark-themed dashboard. We used **Framer Motion** for smooth UI transitions and **React Hot Toast** for real-time status updates.
- **Backend**: **Node.js & Express** serving as the API layer and the static host for the React build. This ensures seamless cookie-based session handling without cross-domain "flakiness."
- **Database**: **Neon Serverless PostgreSQL** for persistent storage of audit logs, user sessions, and verified metrics.
- **Inference Layer**: Direct API integration with **Groq Cloud** for real-time AI-driven auditing.
- **Infrastructure**: Hosted on **Render** (Production environment) with automated GitHub CI/CD pipelines.

## 4. Stability & Production Readiness
The production environment has been hardened with:
- **Proxy Trust (`trust proxy: 1`)**: Correctly handles Render's load balancer headers.
- **SameSite None Cookies**: Ensures identity persistence across subdomains.
- **Rate Limiting**: Protects AI inference and PDF upload endpoints from DOS/brute-force attacks.
